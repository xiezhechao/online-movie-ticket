2\URL地址规则
pathinfo模式
***/index.php/模块/控制器/方法/名/值
兼容模式:
***/index.php?s=模块/控制器/方法/名/值

3\C层编写规则
a\命名 application/index/controller/Xxx.php 
类名和类文件名一致
b\使用命名空间
namespace app\index\controller;
use think\Controller;
c\可以继承父类Controller
d\可以是Controller 提供的方法
e\方法名没有要求

例子:呈现文章详细页
application/index/controller/News.php
namespace app\index\controller;
use think\Controller;
class News extends Controller{
    /**
     * 呈现文章详细页
     */
    function detail(){
	//给模板传值
	$this->assign('t',"今天学习ThinkPHP");
        return $this->fetch([path]);//默认模板调用规则 view/news/detail.html
    }
}

Controller类提供的方法:
给模板传值
assign(名,值)
显示模板
display(模板内容,传所有值)
获取模板的运行结果
fetch("模板路径")
类的初始化,和构造方法作用一样
_initialize()
成功提示跳转
success(提示语,跳转的地址,数据,等待时间)
失败后提示跳转
error(提示语,跳转的地址,数据,等待时间)
跳转(没有提示)
redirect(url,params)
